A number that represents a capability tied to a specific outcome in the active Install Base. This is a complement to the Install Base that will be reaching LDoS.

Architectures Categories Description
Enterprise Networking LAN Metric (TrustSec Readiness): The penetration of the Doppler ASIC platforms in the account's access layer. An account that is 35% TrustSec Ready means 35% of the access layer consists of Catalyst 3650, 3850, or Cat 4500 with the Sup8.

How do I use this metric? Using the example above 35% of the access layer can enable the full TrustSec network segmentation strategy. 35% of the access layer can enable Network As A Sensor with Lancope. Both of these are cross-architecture plays with the Security team. If a customer derives value from that 35% of the network, the remaining 65% of the network needs to be refreshed to derive the same value. This 65% of the network is usually LDoS and is always older generation capability.
802.11ac (Advanced Analytics Readiness): This is the penetration of 802.11ac capability in the network. Advanced capabilities such as the Cisco-Apple relationship, hyperlocation, CMX Analytics, require a modern and robust WLAN. The higher the percentage, the greater the customer’s ability to enable an outcome.

How do I use this metric? Likewise, if a desired outcome is realized on this “Ready” portion of the network, the remainder of the network is now a refresh target. Not because of age or capability; but because of value the customer has realize elsewhere in the network.
iWAN/ISR4K: iWAN/ISR4K: The penetration of the ISR4K and the measure of a WAN’s ability to implement the full iWAN suite of capabilities.
Security Naked ASA: The percentage of the active ASA install base that is current generation ASA (post 2014); but does not have SourceFire attached.
Legacy ASA: The percentage of the active ASA install base that is either reaching LDOS and/or is sold from 2008-2013.
How do I use this metric? An ASA implementation that is 10% Naked ASAs means that 10% of the ASA kit can realize additional value by attaching SourceFire. The remaining ASA kit, and therefore the rest of the customer's Security presence, can only realize that same value with both an ASA AND SourceFire upgrade.
Collaboration The Collaboration Index: The Collaboration Index measures the account's adoption of the broader Collaboration portfolio. The smaller the number, the more heavily leveraged the customer is on dialtone and telephony. The higher the number, the more broadly the customer has invested in our portfolio.

The number is compared to the Theater Average. If it falls below the Theater Average, the Account Team is further inspected on their sales of WebEx, Video, and Contact Center. If any of those are in the active current year Install base or SFDC pipeline, they are flagged green. Any 'red' is the recommended sales motion.

For example, if a customer falls below the theater index and Video, WebEx, and Contact Center are inspected, it is expected that the Account Team lead with Contact Center to broaden out the portfolio.
DCV Modern UCS: Modern UCS: The ratio of M3/M4 to the total UCS install base. The higher the number, the more modern the UCS kit
Modern DCS: Modern DCS: The ratio of latest generation Nexus to the Install Base. The higher the number, the modern modern the Nexus install base.
How do I use this metric? How to use: The penetrations for both ought to be inline with each other. A customer with a 30% Modern UCS footprint and a 90% modern DCS footprint might be lagging in compute capabilities.
Suggested Cross Architecture Plays you can derive from these data: Cross Architecture Play 1: Network Segmentation, Network as a Sensor, Security-Driven Network Refresh: as explained above
Cross Architecture Play 2: If a customer has a high index, and they are interested in the Cisco Apple Strategy, we can look at two things.

a. A high index means they have the majority of the Collaboration pieces to move forward
b. We then look at their 802.11ac Readiness. If that number is low, it means the customer needs to enhance their WLAN to effectively adopt this solution.
Cross Architecture Play 3: If a customer has a high Modern DCS footprint, leading with the Secure ACI / Secure DC strategy can drive security attach.
Cross Architecture Play 4: If a customer has a DC-Cloud strategy and there are multiple branches, the APIC-EM, and SDN strategies impact both the WAN and the DC.
